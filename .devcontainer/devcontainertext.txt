Purpose:
This file tells GitHub Codespaces or VS Code Dev Containers how to build and run

In this case, it creates a Python 3.11 environment, installs your app dependencies,

devcontainer.json supports JSONC, which means comments are allowed in the actual

ORIGINAL DEVCONTAINER FILE WITH COMMENTS

  // This command runs after the repository content is available inside the container.

SIMPLIFIED SUMMARY
==================

This devcontainer.json file does the following:

1. Creates a Python 3.11 Linux development environment.
2. Opens README.md and app.py automatically.
3. Installs Python-related VS Code extensions.
4. Installs Linux packages from packages.txt if that file exists.
5. Installs Python packages from requirements.txt if that file exists.
6. Installs Streamlit.
7. Automatically runs the Streamlit app using app.py.
8. Opens the app preview on port 8501.


OPTIONAL CLEANER VERSION OF updateContentCommand
================================================

The original updateContentCommand works, but it is dense and includes sudo apt upgrade -y,
which can slow down container startup.

A cleaner version would be:

"updateContentCommand": "if [ -f packages.txt ]; then sudo apt update && sudo xargs apt install -y < packages.txt; fi; if [ -f requirements.txt ]; then pip3 install --user -r requirements.txt; fi; pip3 install --user streamlit; echo '✅ Packages installed and Requirements met'"

Why this may be better:

- Easier to read.
- Avoids upgrading the whole Linux environment each time.
- Still installs system packages, Python dependencies, and Streamlit.
